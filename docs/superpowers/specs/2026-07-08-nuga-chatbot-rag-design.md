# nuga-log-app — RAG 챗봇 전환 설계

- 작성일: 2026-07-08
- 대상: `989-alt/nuga-log-app` (Next.js 14 App Router, Vercel, master 브랜치)
- 상태: 승인됨 (브레인스토밍 완료)

## 1. 목표

기존 **정적 폼 기반** 누가기록 생성 앱을 **대화형 RAG 챗봇**으로 전면 교체한다. 챗봇이 스킬(neis-behavior-log)의 인터뷰처럼 상황·조치를 캐물어 필수 사실을 채우고, Korean Law MCP로 핵심 조문과 **판례를 세밀하게 탐색**해 근거를 끌어와, 지금과 동일한 산출물(5블록 + 신설 법적보호분석)을 낸다.

핵심 가치: **변호사 없이도 교사가 누가기록으로 스스로 법적 보호를 할 수 있는 수준**의 근거·분석을 제공하되, NEIS 본문은 스킬 철칙대로 **법률 직접 명시·단정을 하지 않는다**(헤지 서술). 깊은 법적 분석은 본문 밖 교사용 섹션에만 쌓는다. 나아가 **본문 자체의 법적 방어력**을 별도 검증 에이전트가 감사·보강한다.

## 2. 확정된 설계 결정

| # | 결정 | 선택 |
|---|---|---|
| 1 | 기존 폼과의 관계 | **전면 교체** — 챗봇이 유일 진입점 (폼은 타인 사용 이력 없음·불편) |
| 2 | 질문 결정 방식 | **가드레일 있는 에이전트** — LLM이 자유롭게 묻되, 필수슬롯 미충족 시 생성 금지 하드 게이트(코드) |
| 3 | RAG 검색 실체 | **하이브리드** — 핵심 조문은 로컬 KB, **판례는 라이브 MCP 세밀 탐색** |
| 4 | 깊은 법적 분석 위치 | **전용 블록 신설(A) + 온디맨드 심화(C)** — `[법적 보호 분석]` 블록 + "판례 더 찾기" |
| 5 | 본문 방어력 | **법률 검증 에이전트** 추가 — 본문을 금지규칙 + 2021도13926 방어요건으로 감사·보강 |
| 6 | 저장 | **다운로드(.txt) + Chrome 폴더 직접 저장(File System Access) + 복사** |
| 7 | AI 계층 | **BYOK 전용** — Claude·OpenAI·Gemini, 모델 선택 UI에 등급 표시(결제 키 권장), 서버 무저장·무로그 |
| 8 | 대화 상태·프라이버시 | **무상태 서버 + 대화는 브라우저 메모리만** — 최종본만 localStorage 저장 |
| 9 | 특수교육대상 여부 | **사용자가 직접 옵션 선택**(자동 감지 아님) |
| 10 | 특수교육 세부 유형 | **장애 유형 체크박스 목록**(복수 선택), 교사용 섹션 조정에만 사용·본문 미기입(민감정보) |

## 3. 아키텍처와 데이터 흐름

모두 무상태 서버, 대화는 브라우저 메모리에서만 유지한다.

```
[브라우저 챗 UI]  ── transcript+slots+specialEd ──▶  POST /api/chat  (인터뷰 턴)
     ▲                                                  │
     │  다음 질문 / "생성 준비됨"                          ├─ LLM(BYOK) 호출: 다음 질문 + 슬롯 추출
     │                                                  ├─ (모델 요청 시) Law MCP 툴콜 ↔ 서버 중개(라운드 상한)
     │                                                  └─ 코드 게이트: 필수슬롯 미충족→readyToGenerate 강제 false
     │
     │  필수슬롯 충족 + 사용자가 "누가기록 생성" 클릭
     ▼
[생성]  ── slots+transcript+specialEd ──▶  POST /api/generate  (최종 합성)
        ① 근거 검색: 로컬 KB(고시·법률) 항상 + 라이브 MCP(판례 세밀 탐색, 유형 기본검색 1회 보장)
        ② 1차 draft: 5블록 + [법적 보호 분석]
        ③ 법률 검증 에이전트: 본문 감사(금지규칙 + 2021도13926 4요건) → 미달 시 사실 범위 내 보강, 통과까지 루프(상한 2~3)
        ④ 판례 환각 제거: 검색결과에 없는 사건번호 제거
        ⑤ 최종본 반환
     ▼
[결과]  복사 · 다운로드(.txt, 스킬 파일명) · Chrome 폴더 직접 저장 · "판례 더 찾기"(온디맨드)
        └ 최종본만 localStorage 히스토리
```

**엔드포인트**
- `POST /api/chat` — 인터뷰 턴. 입력: `{ messages, slots, caseTypeHint?, specialEd, byok }`. 출력: `{ assistantMessage, caseType, slotUpdates, readyToGenerate }`. 서버 무상태.
- `POST /api/generate` — 기존 확장. 근거검색 → draft → 검증 루프 → 최종 5+1블록.

**에이전트 툴콜**은 서버가 소유한 루프로 처리(제공사 무관). 모델이 `search_law`/`search_decisions`/`get_decision_text`를 요청하면 서버가 `korean-law-mcp-shared` HTTP(JSON-RPC)로 중개해 결과를 회신한다. 지연·비용 보호를 위해 **툴콜 라운드 상한**을 둔다. 로컬 KB grounding은 툴콜과 무관하게 항상 선주입된다.

## 5. 인터뷰 게이트

**진행(스킬 3단계 이식):**
1. 교사 자유 서술 → 봇이 7유형 중 분류(애매하면 한 줄로 확인).
2. 봇이 한 턴에 1~3개씩 필수슬롯을 질문: `일시(연·월·일·요일·교시)`·`장소`·`관찰행동`·`교사발화+학생발화(직접인용 1쌍)`·`지도단계`·`학생반응`·`보호자통보(시각·수단)`·`후속`. 유형별 추가슬롯(학폭=조사정황, 자해=위험도·통보, 아동학대의심=신고의무, 긴급제지=제지시각·보고대상, 민원=교권침해 정황).
3. **하드 게이트(코드)**: 서버 `validateSlots`가 유형별 필수슬롯 검사 → 하나라도 비면 `readyToGenerate=false` 강제 + 봇이 그 슬롯 재질문. 모델이 조기 종료하려 해도 코드가 막는다. 아동학대 의심 감지 시 신고의무 안내를 먼저 노출.
4. 충족되면 "준비됨" → 교사가 생성 클릭.

**특수교육대상**: 사용자가 예/아니오 직접 선택. "예"이면 장애 유형 체크박스(시각·청각·지적·지체·정서행동·자폐성·의사소통·학습·건강장애·발달지체, 복수 선택). 선택값은 `[향후 안전한 지도 방법]`·`[법적 보호 분석]` 맞춤과 `[근거]` 고시 제15조③ 인용에 사용. **본문 자동 삽입 금지**(민감정보).

| `[법적 보호 분석 — 교사용 / NEIS 미붙여넣기]` (신설) | 사안 사실 ↔ 라이브 검색된 실제 판례(사건번호·요지) ↔ 방어 논리. "판례 더 찾기"로 온디맨드 심화 |

## 7. 검색·검증 상세

**근거 검색(`lib/lawRetrieval.ts`)**
- **로컬 KB(항상 선주입)**: 고시 제2026-3호(5단계·제15조③), 초중등교육법 제20조의2, 아동복지법 제17조·제71조, 아동학대처벌법 제2조3호단서·제10조, 판례 2021도13926·2015도13488.
- **라이브 MCP(사안 맞춤 심화)**: `search_law`(고시 현행 번호·시행일) → `search_decisions`(사실·유형 키워드) → `get_decision_text`(상위 판례 요지). 에이전트가 필요 시 호출(라운드 상한) + 유형별 기본 키워드 검색 1회 결정적 보장.
- **폴백**: MCP 타임아웃·실패 시 조용히 로컬 KB만 사용. 출력은 MCP에 막히지 않는다.

**판례 환각 방지(중요)**: `[법적 보호 분석]`·`[근거]`의 판례 사건번호는 MCP가 실제 반환한 것만 인용. 검증 에이전트가 검색결과 집합에 없는 사건번호를 제거한다. 없는 판례 창작 차단.

**법률 검증 에이전트(`buildVerifyPrompt`)**
- 입력: 본문 draft, 확정 사실, 검색된 근거.
- 감사 축:
  - **금지 규칙**: 법률단정·평가어·모욕·비하·민감정보·타인실명 0건(스킬 6·6.5).
  - **방어력(2021도13926)**: ①구체 관찰사실 ②적용 지도단계 명시 ③비례성(단계적·최소개입) ④후속 — 4요건이 본문에 드러나는가.
- 출력 JSON: `{ pass, violations[], missingElements[], revisedBody }`. `pass=false`면 revisedBody 채택 후 재감사(상한 2~3회). **보강은 입력 사실 범위 내에서만**(없는 사실 창작 금지). 초과 시 최선본 + "검토 권장" 경고.

## 8. 에러 처리

- LLM 오류(429/401/404): 기존 `messageForStatus` 재사용, 턴 단위 재시도.
- MCP 실패·타임아웃: 조용히 로컬 KB 폴백, 출력 계속.
- 검증 루프 상한 초과: 최선본 채택 + 경고 표기.
- 모델 JSON 깨짐: 1회 복구 재요청 후 실패 시 안내.
- 게이트가 필수슬롯 미충족 생성을 원천 차단.

## 9. 프라이버시

- transcript는 브라우저 메모리에만(새로고침 소멸). 서버 무저장·무로그(요청 본문 로깅 금지 유지).
- 장애 유형 등 민감정보는 본문 미기입, 클라이언트에만 존재.
- 최종 누가기록만 localStorage(`nuga-log-history-v1`) 저장, 사용자 명시 저장/삭제.
- BYOK 키는 클라이언트(localStorage)만, 서버 저장·로그 안 함.

## 10. 테스트(Vitest, 기존 43개 확장)

- 게이트: 유형별 필수슬롯 판정, 슬롯 추출 파싱.
- 검증 에이전트: 금지어(법률단정·평가어·모욕) 탐지, 방어 4요건 누락 탐지.
- 판례 환각 제거기: 검색결과에 없는 사건번호 제거.
- `lawRetrieval` 병합·폴백, `fileSave` 파일명, 프롬프트 빌더.
- 회귀 픽스처: 2026-07-08 실제 사안 2건(의복정리 주의 / 수업중 제지·발언 지도).

## 12. YAGNI로 제외

- 고전적 벡터 RAG(임베딩·벡터DB): 검색 대상이 좁아 과함. 하이브리드로 충분.
- 서버 세션 저장소·로그인·DB: 프라이버시 원칙과 충돌, 불필요.
- 무료 서버 Gemini 폴백: 에이전트+판례 RAG+검증 루프 워크로드에서 429·품질저하로 실망 유발 → 제거.
